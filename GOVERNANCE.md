# Governance

Kinetik is currently maintained by a small maintainer team led by the project author.

## Roles

- **Maintainer**: reviews and merges PRs, manages releases, sets roadmap priorities
- **Contributor**: opens issues, discussions, and pull requests

## Decision process

- Small fixes/docs: maintainer discretion
- API/design-system changes: discussed in issue/discussion before merge
- Breaking changes: require migration notes in changelog and README updates
